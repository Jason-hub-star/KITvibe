# UI Remediation Lock — 2026-04-09

> 목적: 대시보드/수업선택/채팅 입력/단계 UX 관련 패치를 시작하기 전에 범위와 비범위를 잠근다.

## 왜 지금 잠그는가

이번 묶음은 단순 스타일 수정이 아니라 아래 4가지가 함께 얽혀 있다.

- 교사 대시보드 행동 유도 (`리포트 확인`, 뒤로가기, 히트맵 존재감)
- 앱 전체 색상 토큰 (`--primary`가 너무 검정색)
- 학생/교사 공통 수업 선택 UX (검색 부재)
- 학생 채팅 학습 진행 UX (입력창 정렬, 4단계 의미 전달)

이 상태에서 바로 코드부터 수정하면 범위가 늘어나기 쉬워서, 먼저 패치 목적과 게이트를 문서로 고정한다.

## 현재 확인된 사실

### 1. `리포트 확인` 버튼은 현재 동작이 없다

- 위치: `src/components/teacher/AISpotlightCard.tsx`
- 현재는 `<button>`만 있고 `onClick`, `href`, 라우팅이 없다.
- 따라서 사용자 입장에서는 CTA처럼 보이지만 실제 행동으로 연결되지 않는다.

### 2. 오개념 히트맵 기능은 이미 존재한다

- 위치: `src/components/teacher/MisconceptionHeatmap.tsx`
- 데이터 소스: `getDashboardData()`가 내려주는 `heatmap`
- 대시보드에서 실제 렌더링 위치: `src/app/teacher/dashboard/page.tsx`
- 결론: "기능 없음"이 아니라 "존재하지만 주변 CTA/설명/동선이 약함" 쪽에 가깝다.

### 3. 교사 대시보드 상세 화면에는 이전 페이지로 돌아가는 기능이 없다

- 위치: `src/app/teacher/dashboard/page.tsx`
- `?lesson=...`가 있는 상세 화면에는 현재 뒤로가기/목록 복귀 링크가 없다.
- 학생 쪽은 `/student/ask`와 `/student/summary`에 복귀 동선이 있다.

### 4. 앱 전체 `primary`가 순수 검정이라 체감 대비가 강하다

- 위치: `src/app/globals.css`
- 현재 `--primary: #000000`
- `bg-primary`, `text-primary`, `TableHeader bg-primary`, 학생 말풍선, CTA 카드에 전파된다.
- 결과적으로 `AI 보충 분석` 카드뿐 아니라 앱 전체가 의도보다 더 강하고 딱딱하게 보인다.

### 5. 학생/교사 수업 선택 화면 모두 검색 기능이 없다

- 교사: `src/components/teacher/LessonSelector.tsx`
- 학생: `src/app/student/ask/page.tsx`
- 현재는 최신순 목록만 보여주며 제목/주제 필터링이 없다.

### 6. 학생 채팅 입력창 버튼은 "입력창 안에 들어간 느낌"이 약하다

- 위치: `src/components/student/ChatInput.tsx`
- 구조상 absolute 버튼이 textarea 위에 얹혀 있으나, 시각적으로는 독립 버튼처럼 보일 수 있다.
- 사용자 피드백상 "입력창 밖으로 빠져나와 있다"는 인상이 확인됐다.

### 7. 4단계 진행은 숫자는 보이지만 의미가 충분히 드러나지 않는다

- 헤더: `src/components/student/QuestionChat.tsx`
- 보조 진행 표시: `src/components/student/SessionStats.tsx`
- 프롬프트 정의: `src/lib/prompts/index.ts`
- 현재 사용자에게는 `1/4단계` 숫자만 강하게 보이고,
  어떤 단계인지, 언제 다음 단계로 넘어가는지, 마지막 보상이 무엇인지가 UI에 직접 드러나지 않는다.

## 이번 패치에서 잠그는 결정

### A. `리포트 확인`은 "새 페이지"가 아니라 "현재 대시보드 내 상세 영역 이동"으로 잠근다

- 이번 범위에서는 별도 `report` 페이지를 만들지 않는다.
- `리포트 확인` 버튼은 현재 페이지의 `Question Log` 또는 리포트 섹션으로 스크롤/이동시키는 CTA로 바꾼다.
- 이유: SSOT에 교사용 별도 리포트 페이지는 없고, 이번 요구는 동선 보강이 핵심이다.

### B. 오개념 히트맵은 "신규 기능"으로 만들지 않는다

- 이미 존재하는 기능으로 간주한다.
- 이번 패치에서는 히트맵 노출 품질과 주변 설명/동선을 개선한다.
- 데이터 집계 로직 자체는 이번 범위에서 변경하지 않는다.

### C. 교사 상세 대시보드에는 "목록으로 돌아가기"를 명시적으로 추가한다

- 브라우저 history 의존 대신 `/teacher/dashboard`로 복귀하는 링크/버튼을 둔다.
- 이유: 링크 공유 진입, 새 탭 진입에서도 일관되게 동작해야 한다.

### D. 앱 전체 primary는 Obsidian식 graphite 계열로 완화한다

- 순수 검정 `#000000`을 기본 primary로 유지하지 않는다.
- 기준 후보는 Obsidian neutral `#222222`
- 적용 범위: global token 우선
- 비범위: 이번 패치에서 별도 다크모드/복수 테마는 도입하지 않는다.

### E. 수업 선택 검색은 학생/교사 공통 컴포넌트로 잠근다

- 검색은 서버 검색이 아니라 클라이언트 필터링으로 구현한다.
- 필터 대상은 `title`, `topic`, `subject`
- 결과 없음 상태를 명시한다.
- 이유: 현재 lesson 수가 적고, UX 요구는 탐색성 개선이지 검색 인프라가 아니다.

### F. 채팅 입력창은 "하나의 입력 셸 안에 버튼이 포함된 구조"로 재구성한다

- 이미지 버튼과 전송 버튼이 텍스트 입력과 동일 border box 안에 들어가도록 재구성한다.
- 단순 위치 미세조정보다 구조 변경을 우선한다.

### G. 4단계 로직은 이번 패치에서 "행동 규칙 유지 + 의미 노출 강화"로 잠근다

- 단계 수는 계속 4단계로 유지한다.
- 이번 패치에서 단계 판정 알고리즘 자체를 바꾸지 않는다.
- 대신 다음을 UI로 명확히 드러낸다.
  - 현재 단계 이름
  - 다음 단계 이름
  - 언제 이동하는지
  - 4단계 완료 후 무엇이 열리는지

## 현재 단계 진행 규칙 정리

### Grill-Me

- `wrong`이면 단계 유지
- `partial` 또는 `correct`이면 다음 단계로 이동 가능
- 연속 `wrong` 3회면 `Guide-Me`로 자동 전환

### Guide-Me

- 설명 모드에서 `correct`가 나오면 다시 `Grill-Me`로 복귀
- 복귀 시 다음 단계로 1칸 전진 가능

### Quick-Me

- 사용자가 직접 고르거나 긴급 표현으로 자동 전환되는 빠른 풀이 모드
- `답만`, `빨리`, `시간 없어`, `바로 풀어줘` 입력은 `Quick-Me` 자동 전환을 허용한다

### 단계 완료 후 보상

- `4/4` 도달 시 미니퀴즈가 열린다
- 미니퀴즈 통과 시 현재 UX 보상은 `오개념 회복` 문구다
- 이번 패치에서는 강한 게임화 보상은 추가하지 않는다
