# Mobile Text Lock — 2026-04-10

> 목적: 모바일 기준에서 한글/영문 혼용, 강제 줄바꿈, 메타 정렬 방식이 제각각인 문제를 수정하기 전에 텍스트 규칙을 먼저 잠근다.

## 왜 지금 잠그는가

- 현재 모바일 화면에서 한글과 영어 라벨의 우선순위가 섞여 있다.
- 일부 텍스트는 `<br />`에 의존해 폭이 달라도 같은 줄바꿈을 강제한다.
- 수업 선택 카드처럼 좌/우 메타가 갈라진 구조는 모바일에서 시선 흐름을 끊을 수 있다.
- 이 문제는 단순 색상 토큰이 아니라 `콘텐츠 규칙 + 타이포 규칙 + 레이아웃 규칙`이 함께 걸려 있다.

## 현재 확인된 사실

### 1. 랜딩의 히어로 본문은 강제 줄바꿈을 사용한다

- 위치: `src/app/page.tsx`
- 문제: `<br />`가 모바일 폭에서도 같은 위치에서 줄을 끊어 자연스러운 재흐름을 막는다.

### 2. 섹션 라벨의 언어 우선순위가 섞여 있다

- 위치: `src/app/page.tsx`, `src/components/layout/AiRuntimeReadiness.tsx`, `src/components/student/SessionSummaryView.tsx`
- 예: `Get Started / 시작하기`, `How it works / 이렇게 동작해요`, `Session Summary`
- 문제: 어떤 화면은 영어가 먼저, 어떤 화면은 한국어가 먼저라 앱 전체 톤이 흔들린다.

### 3. 수업 선택 카드의 메타가 모바일에서 별도 열처럼 보인다

- 위치: `src/components/lesson/LessonSearchList.tsx`
- 문제: 제목/주제는 좌측, 과목/날짜는 우측 고정이라 좁은 화면에서 정보 흐름이 분리된다.

### 4. 학생 요약 화면은 한국어 본문인데 상단 라벨만 영어다

- 위치: `src/components/student/SessionSummaryView.tsx`
- 문제: 정보는 잘 보이지만 언어 톤이 통일되지 않는다.

## 이번 패치에서 잠그는 규칙

### A. 모바일 UI는 한국어 우선으로 잠근다

- 모바일에서 섹션 라벨, 헤더, CTA는 한국어를 기본으로 쓴다.
- 영어는 기술명, 모델명, 보조 캡션에서만 제한적으로 사용한다.
- `OpenAI`, `Gemma`, `runtime` 같은 기술명은 필요한 곳에만 남긴다.

### B. 강제 줄바꿈 대신 자연 줄바꿈으로 잠근다

- 본문 카피에서 `<br />`로 줄을 고정하지 않는다.
- 반응형 텍스트 크기, 최대 폭, line-height로 자연스럽게 줄바꿈되게 만든다.

### C. 모바일 메타 정보는 세로 흐름 우선으로 잠근다

- 모바일에서는 제목 → 주제 → 과목/날짜 순으로 쌓이는 구조를 우선한다.
- `justify-between`으로 좌/우를 강하게 갈라놓는 구조는 `sm` 이상에서만 사용한다.

### D. 섹션 마이크로 라벨은 semantic utility로 잠근다

- `ui-kicker`: 섹션 라벨, eyebrow, 필터 버튼
- `ui-micro`, `ui-micro-strong`: 날짜, 상태, 작은 메타
- `ui-support`: 작은 설명 문장
- 라벨 문구 자체는 한국어 우선으로 통일한다.

### E. 이번 범위는 랜딩/수업선택/학생 요약만 고친다

- 랜딩: 히어로 카피, 섹션 라벨, Gemma-ready 카드 라벨
- 수업 선택: 카드 메타 배치
- 학생 요약: 상단 라벨, 제목 스케일, CTA 정렬
- 학생 채팅: 미니퀴즈/보조 라벨 한국어 정리
- 교사 대시보드: 질문 로그/보조 라벨 한국어 정리

## 이게 디자인 토큰인가

- 글자 크기, 행간, 간격, 색상은 디자인 토큰 영역이다.
- 하지만 어떤 언어를 먼저 보여줄지, 어디서 줄을 끊을지, 모바일에서 메타를 아래로 내릴지는 콘텐츠/컴포넌트 규칙이다.
- 결론: 이번 패치는 `디자인 토큰 + 콘텐츠 규칙`을 함께 만지는 범위로 본다.
