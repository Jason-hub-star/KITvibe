## A. 랜딩 페이지 (Landing)

### TC-01: 역할 선택 → 교사 흐름 진입
- **사전조건:** localStorage 비어있음 (역할 미선택)
- **단계:**
  1. `localhost:3000` 접속
  2. "교사로 시작 →" 카드 클릭
- **기대 결과:**
  - POST `/api/users` 호출 → 교사 유저 생성
  - localStorage `pulda-user`에 `{ role: "teacher", userId, userName }` 저장
  - `/teacher/upload` 페이지로 이동
  - Header에 "교사" 배지 + 선생님 이름 표시
- **우선순위:** P0

### TC-02: 역할 선택 → 학생 흐름 진입
- **사전조건:** localStorage 비어있음
- **단계:**
  1. `localhost:3000` 접속
  2. "학생으로 시작 →" 카드 클릭
- **기대 결과:**
  - POST `/api/users` 호출 → 학생 유저 생성
  - `/student/ask` 페이지로 이동 (수업 선택 UI)
- **우선순위:** P0

### TC-03: 역할 전환 (교사 → 홈 → 학생)
- **사전조건:** 교사로 로그인된 상태

  2. 랜딩 페이지 도착 확인

## B. 교사 업로드 (Teacher Upload)

### TC-05: PDF 업로드 → 지식베이스 생성 (Happy Path)
- **사전조건:** 교사로 로그인, 이차방정식 PDF 준비
- **단계:**
  1. `/teacher/upload` 접속
  2. 제목 입력: "이차방정식의 근의 공식"
  3. 주제 입력: "이차방정식, 판별식"
  4. PDF 파일 드래그앤드롭 또는 클릭 업로드
  5. "지식베이스 생성하기" 클릭
- **기대 결과:**
  - 파일 목록에 파일명 + 크기 표시
  - 버튼 클릭 시 "처리 중..." 표시
  - 성공 Alert: "지식베이스 생성 완료!" + "대시보드 보기" + "링크 복사"
  - 입력 필드 disabled 처리
  - Supabase `lessons` + `lesson_materials` 테이블에 행 삽입 확인
- **우선순위:** P0

### TC-06: 업로드 폼 유효성 검증
- **사전조건:** 교사로 로그인
- **단계:**
  1. 제목 비어있는 상태에서 파일만 업로드
  2. 파일 없이 제목만 입력
  3. 10MB 초과 파일 드롭
  4. 지원하지 않는 형식(.exe) 드롭
- **기대 결과:**
  - 제목 없음: "지식베이스 생성하기" 버튼 disabled
  - 파일 없음: 버튼 disabled
  - 10MB 초과: toast 에러 메시지
  - .exe: toast 에러 "파일 형식 미지원"
- **우선순위:** P1

### TC-07: 링크 복사 → 학생 접속 연결
- **사전조건:** TC-05 완료 상태 (지식베이스 생성 완료)
- **단계:**
  1. "링크 복사" 버튼 클릭
  2. 클립보드 내용 확인
  3. 해당 URL로 접속 (학생 역할)
- **기대 결과:**
  - 클립보드: `{origin}/student/ask?lesson={lessonId}` 형식
  - toast: "학생 링크가 복사되었습니다!"
  - 해당 URL 접속 시 채팅 UI로 바로 진입
- **우선순위:** P1

## C. 학생 질문 채팅 (Student Chat)

### TC-08: 수업 선택 → 채팅 진입 → 질문 전송 (Happy Path)
- **사전조건:** 학생으로 로그인, 수업 1개 이상 존재
- **단계:**
  1. `/student/ask` 접속
  2. 수업 카드 클릭
  3. 채팅 UI 진입 확인
  4. "이차방정식 판별식이 뭐야?" 입력 후 Enter
- **기대 결과:**
  - 수업 선택 화면에 수업 카드 목록 표시
  - 클릭 시 채팅 UI (헤더: 수업명 + "질문 1/4")
  - 초기 안내 메시지: "안녕! {수업명}에 대해 궁금한 점이 있으면..."
  - 메시지 전송 시 user 버블 → assistant 버블(스트리밍) 표시
  - 스트리밍 중 bounce 인디케이터 표시
  - AI 응답 완료 후 `[RECOMMENDATION]` 등 태그가 제거된 깨끗한 텍스트
- **우선순위:** P0

### TC-09: studentId 전달 검증 (P0 버그 수정 확인)
- **사전조건:** 학생으로 로그인 (localStorage에 `pulda-user` 존재)
- **단계:**
  1. 수업 채팅 진입
  2. 질문 전송
  3. 네트워크 탭에서 POST `/api/questions` 요청 확인
- **기대 결과:**
  - request body의 `student_id`가 빈 문자열이 아님
  - `student_id`가 localStorage의 `pulda-user.userId`와 일치
  - API 응답 `{ success: true }`
  - ~~이전 버그: `pulda_user_id` 키로 읽어서 항상 빈 문자열 → 400 에러~~
- **우선순위:** P0

### TC-10: 모드 전환 (Grill-Me → Guide-Me → Quick-Me)
- **사전조건:** 채팅 진입 상태
- **단계:**
  1. 기본 모드 "GRILL-ME" 확인
  2. "GUIDE-ME" 탭 클릭
  3. 질문 전송
  4. "QUICK-ME" 탭 클릭
  5. 질문 전송
- **기대 결과:**
  - 모드 탭: active 상태 underline이 border와 겹침 (수정된 ModeSelector)
  - Guide-Me: 단계별 설명 톤의 응답
  - Quick-Me: 간결한 풀이 톤의 응답
  - 각 모드 전환 시 스트리밍 중이 아니면 즉시 전환
  - 스트리밍 중: 모드 버튼 disabled
- **우선순위:** P1

### TC-11: 학생 채팅에서 홈 복귀 (P1 수정 확인)
- **사전조건:** 학생으로 채팅 진입 상태

## E. 크로스커팅 (접근성 / 반응형 / 에러)

### TC-16: 키보드 네비게이션 (접근성)
- **사전조건:** 없음
- **단계:**
  1. 랜딩 페이지에서 Tab 키로 포커스 이동
  2. 역할 카드에서 Enter/Space로 선택
  3. 업로드 폼에서 Tab 순서 확인 (제목→주제→파일→버튼)
  4. Footer 링크에 Tab 도달 가능 여부
  5. 채팅 입력에서 Enter→전송, Shift+Enter→줄바꿈
- **기대 결과:**
  - 모든 인터랙티브 요소에 Tab 도달 가능
  - Footer: `<button>` 태그로 포커스 가능 (이전 `<span>` 수정)
  - UploadForm: label `htmlFor`↔input `id` 연결됨
  - ChatInput: Enter=전송, Shift+Enter=줄바꿈
  - 삭제 버튼: `aria-label="파일 제거"` 존재
- **우선순위:** P1

### TC-17: 모바일 반응형 (375px)
- **사전조건:** 뷰포트 375×812 설정
- **단계:**
  1. 랜딩: Hero 텍스트 잘림 확인
  2. 학생 수업 카드: 패딩 확인 (`p-4` 적용됨)
  3. 교사 대시보드: QuestionLogTable 가로 스크롤 확인
  4. 채팅: 모드 선택기 + 입력 영역 정상 배치
- **기대 결과:**
  - Hero: `text-4xl` (오버플로 없음)
  - 수업 카드: `p-4` (모바일) / `p-8` (데스크톱)
  - 테이블: `overflow-x-auto` → 가로 스크롤 가능
  - 채팅 하단바: 모드 탭 + 입력 필드가 화면 내 수용
- **우선순위:** P2

### TC-18: 네트워크 에러 시 사용자 피드백
- **사전조건:** DevTools에서 네트워크 오프라인 설정
- **단계:**
  1. 역할 선택 시도 → API 실패
  2. 수업 자료 업로드 시도 → 업로드 실패
  3. 채팅 질문 전송 → 스트리밍 실패
- **기대 결과:**
  - 역할 선택: toast "네트워크 오류가 발생했습니다..."
  - 업로드: toast "수업 생성 중 오류가 발생했습니다."
  - 채팅: assistant 버블에 "죄송해요, 응답 중 오류가 발생했어요. 다시 질문해주세요."
  - 어떤 경우에도 스택 트레이스나 기술 에러 메시지 노출 금지
- **우선순위:** P1
