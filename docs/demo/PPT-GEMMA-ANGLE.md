# PPT Gemma Angle

> 목적: 풀다 AI가 단순히 OpenAI 데모가 아니라, Gemma 4 연동 확장성을 이미 가진 구조라는 점을 발표에서 짧고 강하게 설명하기 위한 문구 모음

## 한 줄 메시지

- 공모전 데모는 OpenAI로 안정성을 확보했고, 제품 구조는 Gemma 4 로컬 연동까지 이미 열어뒀습니다.
- 풀다 AI의 가치는 모델 하나가 아니라, `질문 흐름 + 교사 자료 RAG + 하이브리드 엔진 구조`에 있습니다.
- 같은 학습 흐름을 클라우드와 로컬 AI 양쪽에 연결할 수 있다는 점이 확장성의 핵심입니다.

## 발표용 헤드라인 후보

- 안정적인 데모 위에, Gemma 4 로컬 확장성까지 준비했습니다
- 풀다 AI는 OpenAI 데모가 아니라, Gemma 4-ready 학습 엔진입니다
- 질문 흐름은 그대로, 엔진은 OpenAI와 Gemma 4를 모두 연결할 수 있습니다

## 슬라이드 1장 요약 구조

### 헤드라인

안정적인 데모, 그리고 Gemma 4 확장성

### 본문 3포인트

- 현재 데모는 OpenAI로 안정적으로 동작합니다
- 튜터 엔진은 task 단위로 Gemma 4 로컬 모델로 전환할 수 있게 설계했습니다
- 그래서 비용 절감, 데이터 로컬 처리, 오프라인 확장 가능성을 동시에 확보했습니다

### 하단 보조 문구

- 지금: 심사용 안정성
- 다음: 로컬 AI 비용 절감
- 장기: 학교/기관 단위 온프레미스 확장

## 꼭 지켜야 할 표현

- `Gemma 4-ready`, `로컬 연동 가능`, `task별 전환 가능`
- `현재 데모 기본값은 OpenAI`
- `로컬 모델 전면 배포는 추가 검증 범위`

## 피해야 할 표현

- 학생 폰에서 이미 완벽하게 돌아간다고 단정
- 전면 온디바이스 배포가 준비됐다고 과장
- Gemma가 현재 모든 기능에서 OpenAI보다 우수하다고 단정

## 스크린샷 추천

- 랜딩의 `Runtime / Gemma Ready` 카드
- tutor runtime env 전환 확인 (`npm run ai:runtime-check`)
- 같은 제품 플로우를 유지한 채 OpenAI/Gemma 전환이 가능한 구조도
