# PPT Prompt — 풀다 AI

> 목적: 현재 문서와 실제 구현 상태를 기준으로, 과장 없이 바로 사용할 수 있는 발표자료 생성 프롬프트

## 사용 방식

- Gamma / Canva / Google Slides AI / GPT 기반 deck generator에 그대로 붙여 넣는다.
- 반드시 `현재 구현된 것`과 `확장 준비된 것`을 구분해서 말하게 한다.
- `Gemma 4-ready`는 강조하되, 학생 기기 전면 배포가 이미 끝난 것처럼 말하지 않게 한다.

## 복붙용 프롬프트

```text
다음 조건으로 12~13장 발표용 피치덱 초안을 만들어줘.

프로젝트명: 풀다 AI
한 줄 정의: 수업 후 학생의 막힘 지점을 AI 질문 흐름으로 풀어주고, 교사에게는 오개념 대시보드를 제공하는 학습 보조 솔루션
발표 맥락: KIT 바이브코딩 공모전 제출용 발표 자료
톤: 교육 문제를 명확하게 짚고, 실제 구현과 확장성을 신뢰감 있게 보여주는 톤

절대 규칙:
- 현재 구현된 것과 확장 준비된 것을 구분해서 써라
- 없는 수치, 없는 도입 사례, 없는 사용자 수를 만들지 마라
- 학생 개인 폰에 Gemma 4가 이미 완벽 배포된 것처럼 쓰지 마라
- 풀 LMS, 학교 행정, 학부모 서비스까지 이미 하는 것처럼 쓰지 마라
- OpenAI 데모와 Gemma 4-ready 구조를 동시에 설명하되, "현재 기본 배포본은 OpenAI"라는 점을 숨기지 마라
- 데크는 “모델 자랑”보다 “학습 흐름 + 교사 가치 + 확장 구조” 중심으로 써라
- 기술 스택은 나열만 하지 말고, 왜 그렇게 골랐는지와 어떤 시행착오 끝에 그렇게 잠갔는지 설명하라

디자인 방향:
- 오프화이트 배경, 차콜 텍스트, 민트 포인트
- 흔한 보라색 AI 덱 금지
- 슬라이드당 메시지 1개
- 긴 문단보다 헤드라인 + 3포인트 + 시각자료 지시 구조
- 비교표에서는 핵심 열만 민트 강조

현재 프로젝트의 사실:
- 페이지는 5개: 랜딩, 교사 업로드, 학생 질문, 교사 대시보드, 결과 요약
- 핵심 데모 플로우는 실제로 동작한다:
  교사 업로드 → 학생 질문 → Quick-Me/Grill-Me/Guide-Me → 미니퀴즈 → 세션 요약 → 교사 대시보드
- 입력은 텍스트 + 이미지 1장
- 교사 자료는 PDF 위주이며 Markdown도 가능
- 교사 대시보드에는 질문 TOP 5, 막힌 개념 TOP 5, 오개념 heatmap, AI 보충 분석이 있다
- 학생 모드는 3개다:
  Grill-Me 기본 질문 사다리
  Guide-Me 3회 오답 시 직접 설명
  Quick-Me 긴급 표현 감지 시 빠른 풀이 + 최종 답 즉시 공개
- 현재 배포본 기본 AI는 OpenAI gpt-4o-mini
- 제품 구조는 Gemma 4 계열 로컬 연동이 가능하도록 task별 전환 구조를 갖췄다
- tutor / intent / summary / quiz를 task별로 분리 전환할 수 있다
- 로컬 runtime 점검 스크립트와 OpenAI ↔ Gemma 비교 스크립트가 있다
- Gemma 4-ready는 “연동 가능성과 확장성”의 증거이지, 학생 기기 전면 온디바이스 완료 선언이 아니다
- 기술 선택 관련 시행착오도 실제로 있었다:
  - 공모전 데모 안정성을 위해 기본 배포는 OpenAI로 유지했다
  - 장기 비용과 프라이버시를 위해 Gemma 4-ready 구조를 병행 설계했다
  - Vercel 업로드 한계 때문에 서버 multipart 업로드를 버리고 signed upload URL + direct upload로 바꿨다
  - Quick-Me가 답을 바로 주지 않던 문제 때문에 모드별 프롬프트 분리, 빠른답변 캐시, lesson context cache를 넣었다
  - 문서와 구현 불일치가 있어 SSOT를 먼저 정리한 뒤 기능을 맞췄다

핵심 메시지:
1. 학생은 수업 후 막히지만, 질문을 정리하지 못한다
2. 기존 AI는 정답은 빠르지만 이해를 남기지 못한다
3. 풀다 AI는 교사 자료 기반 RAG + 질문 사다리 + 교사 대시보드를 묶는다
4. 그래서 학생 개입과 교사 개입이 동시에 가능하다
5. 현재 데모는 OpenAI로 안정성을 확보했고, 구조는 Gemma 4-ready라서 비용·프라이버시·오프라인 확장 가능성이 있다

슬라이드 구성:

1. 타이틀
- 헤드라인: 정답을 주는 AI가 아니라, 이해를 만드는 AI
- 서브: 풀다 AI — 수업 후 막힘 해결과 오개념 개입을 연결하는 학습 보조 솔루션
- 하단: KIT 바이브코딩 공모전 / 김주영 / 2026.04
- 시각자료: 랜딩 화면 홀더

2. 문제 정의
- 헤드라인: 학생은 수업이 끝난 뒤에 막히고, 교사는 그 순간을 놓칩니다
- 포인트:
  학생은 무엇을 모르는지 질문으로 정리하기 어렵다
  기존 AI는 정답을 빨리 주지만 이해를 남기지 못한다
  교사는 누가 어디서 막히는지 뒤늦게 알게 된다
- 시각자료: 3개 문제 카드

3. 시장/경쟁 갭
- 헤드라인: 기존 제품은 하나씩 잘하지만, 수업 후 개입 흐름은 비어 있습니다
- 비교축:
  학생 질문 해결
  교사 자료 근거
  질문 사다리
  교사용 오개념 대시보드
  수업 후 보충 개입
- 비교대상:
  풀다 AI / 일반 AI 챗봇 / 문제풀이 앱 / LMS형 서비스
- 강조: 풀다 AI 열

4. 솔루션 한 장 요약
- 헤드라인: 교사의 자료를 기반으로, 학생에게는 질문을 던지고 교사에게는 개입 지점을 보여줍니다
- 포인트:
  수업자료 업로드 → 지식베이스 생성
  학생 질문 → Grill-Me 질문 사다리
  교사 대시보드 → 오개념 heatmap + AI 보충 분석
- 시각자료: 전체 서비스 흐름도

5. 학생 경험
- 헤드라인: 답을 바로 주지 않고, 스스로 풀 수 있게 질문합니다
- 포인트:
  1단계 접근법
  2단계 핵심 개념
  3단계 유사 적용
  4단계 풀이 설명
- 하단: 막히면 Guide-Me, 급하면 Quick-Me
- 시각자료: 학생 채팅 화면

6. 교사 경험
- 헤드라인: 교사는 개별 질문을 넘어서 공통 막힘을 한눈에 봅니다
- 포인트:
  질문 TOP 5
  막힌 개념 TOP 5
  오개념 heatmap
  AI 보충 분석
- 시각자료: 교사 대시보드 화면

7. 실제 구현 증거
- 헤드라인: 기획이 아니라, 이미 끝까지 작동합니다
- 포인트:
  교사 업로드
  학생 질문
  미니퀴즈
  세션 요약
  교사 대시보드
- 하단 문구:
  Production 배포 완료
  실사용 완주 테스트 PASS
- 시각자료: 플로우 캡처 3~4장

8. Quick-Me / Guide-Me / Grill-Me
- 헤드라인: 학생 상태에 따라 개입 강도를 바꿉니다
- 포인트:
  Grill-Me는 질문으로 사고를 유도
  Guide-Me는 3회 오답 시 직접 설명
  Quick-Me는 긴급 표현 감지 시 빠른 풀이와 최종 답 공개
- 시각자료: 모드 비교 카드

9. Gemma 4-ready
- 헤드라인: 안정적인 데모 위에, Gemma 4 로컬 확장성까지 준비했습니다
- 포인트:
  현재 기본 배포본은 OpenAI로 안정적으로 동작
  tutor / intent / summary / quiz를 task별로 분리 전환 가능
  Gemma 4 연동으로 비용 절감, 데이터 로컬 처리, 오프라인 확장 가능성 확보
- 하단 주의 문구:
  학생 기기 전면 온디바이스 배포는 추가 검증 범위
- 시각자료: Runtime / Gemma Ready 카드 또는 구조도

10. 왜 이 구조가 중요한가
- 헤드라인: 모델보다 중요한 것은 학습 흐름을 유지한 채 엔진을 바꿀 수 있다는 점입니다
- 포인트:
  제품 가치는 질문 흐름 + RAG + 교사 개입 구조에 있다
  OpenAI 데모와 Gemma 확장을 같은 흐름 위에 올릴 수 있다
  그래서 심사 단계와 실제 서비스 확장 단계를 모두 설명할 수 있다

11. 왜 이 스택을 골랐는가
- 헤드라인: 기술 스택도 발표용 장식이 아니라, 시행착오 끝에 닫힌 선택입니다
- 포인트:
  Next.js + Supabase는 업로드, 질문, 대시보드, 배포를 가장 빠르게 한 흐름으로 묶기 쉬웠다
  OpenAI는 공모전 데모 안정성을 위해 선택했고, Gemma 4-ready 구조는 장기 비용과 로컬 처리 가능성을 위해 병행했다
  업로드 병목, Quick-Me 품질, SSOT 불일치 같은 문제를 수정하며 현재 구조로 잠갔다
- 시각자료: `선택 / 이유 / 버린 대안` 3열 카드

12. 확장 로드맵
- 헤드라인: MVP는 닫혔고, 다음은 현장 확장입니다
- 포인트:
  로컬 Gemma 품질 비교와 하이브리드 라우팅 고도화
  학교/학원 환경용 온프레미스 또는 기관 서버 모델
  교사 워크플로우 연동과 리포트 확장
- 주의:
  미래 계획은 연한 톤으로 표현

13. 마무리
- 헤드라인: 정답을 빨리 주는 것보다, 이해를 남기는 것이 더 중요합니다
- 포인트:
  풀다 AI는 수업 후 막힘을 학생과 교사 모두에게 보이게 만든다
  실제 데모는 이미 동작한다
  구조는 Gemma 4-ready라서 발표 이후 확장 이야기까지 자연스럽게 이어진다
- 시각자료: 배포 URL, 발표자 정보, 제품 캡처

출력 형식:
- 각 슬라이드를 `슬라이드 번호 / 제목 / 핵심 메시지 / 넣을 문구 / 시각자료 지시`로 정리
- 문구는 발표 슬라이드용으로 짧고 강하게
- 비교표는 발표 가독성을 위해 행/열 수를 최소화
- 현재 구현과 확장 준비를 섞지 말고 구분해서 써라
```

## 이 프롬프트의 핵심 수정 포인트

- `Gemma 4-ready`를 넣되, `현재 기본 배포본은 OpenAI`를 숨기지 않음
- `학생 폰 전면 배포` 같은 과장을 막음
- `Quick-Me / Guide-Me / Grill-Me`, `미니퀴즈`, `세션 요약`, `교사 대시보드` 등 실제 구현된 흐름만 강조
- `왜 이 스택을 골랐는지`와 `어떤 시행착오를 거쳤는지`를 별도 슬라이드로 설명하게 고정
- 발표자료가 `기술 자랑`보다 `제품 가치 + 확장성`을 설명하도록 고정
