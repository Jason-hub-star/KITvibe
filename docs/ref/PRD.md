# PRD — MathTutor AI

> 원본: overview.md §B, §C

## 제품 정의

**한 줄:** 수업 후 학생의 막힘 지점을 AI로 풀어주고, 교사에게는 오개념 대시보드를 제공하는 학습 보조 솔루션

**피치:** "교사의 수업자료에 근거해 학생의 막힘 지점을 힌트 중심으로 해결하고, 교사에게는 오개념 heatmap과 즉시 개입용 보충과제를 제공하는 수업 후 보충 AI 코치."

## MVP 범위

| DO | DON'T |
|----|-------|
| 학생 질문 해결 | LMS 전체 |
| 힌트형 학습 보조 | 학교 행정 |
| 교사용 오개념 파악 | 전 과목 만능 챗봇 |
| 수업 후 보충 개입 | 성적처리, 과금, 학부모 |

## 타겟

- **MVP:** 교사 1명 + 학생 1명 (데모)
- **과목:** 수학

## 문제 정의

1. 학생은 수업 후 막히지만 질문을 정리하기 어렵다
2. 학생은 정답만 보고 넘어가서 진짜 이해가 안 된다
3. 교사는 누가 어디서 막히는지 한눈에 보기 어렵다
4. 교사는 같은 설명을 반복하느라 시간이 많이 든다

## 사용자 흐름

### 교사 플로우

1. 수업 자료 업로드 (PDF/이미지/텍스트) → 텍스트 추출
2. 학습 주제/핵심 개념 입력 → RAG 지식베이스 구축
3. 학생 질문 로그 확인 → 의도 분류 표시
4. 오개념 대시보드 확인 → 히트맵 + TOP 5
5. 보충 과제 참고 → 요약 + 추천 설명

### 학생 플로우

1. 질문 입력 (텍스트/사진) → 의도 자동 분류
2. AI 응답 수신 → 힌트/개념설명/유사문항/답안검토
3. 추가 질문 → 맥락 유지 심화
4. 미니 퀴즈 → 이해도 확인
5. 세션 종료 → 학습 요약

## 입출력

**입력:** 텍스트 + 이미지 1장 (음성/영상 제외)

**학생 출력:** 힌트, 개념 설명, 오답 피드백, 유사문항, 미니퀴즈, 세션 요약
**교사 출력:** 질문 TOP 5, 막힌 개념 TOP 5, 질문 예시, 보충 요약

## 우선순위

| 등급 | 항목 |
|------|------|
| P0 | 수업자료 업로드, RAG 검색, 4 intent 분류, 오개념 대시보드 |
| P1 | 사진 업로드, 미니퀴즈, 세션 요약 |
| P2 (버림) | 풀 LMS, 전과목, 자동 채점 |

## 성공 조건

- [ ] 학생이 질문하고 응답을 받을 수 있다
- [ ] 교사가 막힌 개념을 볼 수 있다
- [ ] AI 활용 이유가 명확하다
- [ ] 데모에서 가치가 바로 보인다
