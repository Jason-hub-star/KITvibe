# Responsive Guide — 풀다 AI

> 모바일/태블릿/데스크톱 반응형 구현과 QA의 기준 문서.
> 디자인 토큰은 `docs/design/DESIGN-TOKENS.md`, 톤/스타일은 `docs/ref/STYLE-GUIDE.md`를 따른다.

## 목적

- 반응형 품질을 “대충 괜찮아 보임”이 아니라 **브레이크포인트별 합격 기준**으로 판단한다.
- MVP 5개 페이지에서 모바일/태블릿 검수 포인트를 통일한다.
- 고정 헤더/하단 입력바가 있는 화면(P-003)에서 잘림, 가림, 터치 불편을 방지한다.

## 기준 뷰포트

| 구분 | 기준 폭 | 대표 기기 | 용도 |
|------|--------|----------|------|
| 모바일 | `390px` | iPhone 12/13/14 | 최소 실사용 기준 |
| 태블릿 | `768px` | iPad portrait | 중간 레이아웃 전환 |
| 데스크톱 | `1280px+` | 노트북 | 최대 폭, 정보 밀도 확인 |

## 공통 원칙

1. **가로 스크롤 금지** — 본문 영역에서 `overflow-x`가 생기면 실패.
2. **텍스트 잘림 금지** — 제목/버튼/탭/라벨은 의미 단위가 잘리지 않아야 한다.
3. **고정 UI 안전영역 확보** — fixed header/footer가 본문 입력, 버튼, 마지막 메시지를 가리면 실패.
4. **터치 우선** — 모바일 주요 클릭 타깃은 최소 `44x44px` 확보.
5. **1열 → 2열 → 다열** — 모바일 1열, 태블릿 2열 이상, 데스크톱에서만 넓은 다열 허용.
6. **토큰 우선** — 반응형 대응도 임의 인라인 스타일보다 토큰/유틸 클래스 조합으로 해결한다.

## 브레이크포인트 규칙

### 모바일 (`< md`)

- 페이지 좌우 패딩: `px-4 ~ px-6`
- 제목은 `text-3xl ~ text-5xl` 범위에서 줄바꿈 허용
- 긴 헤드라인은 `text-[3.5rem]` 고정 대신 `text-4xl sm:text-5xl` 또는 `clamp()` 사용
- 카드/리스트/폼은 기본 1열
- fixed bottom bar가 있으면 본문 `pb-40` 이상 확보
- 입력바 내부 요소는 한 줄 유지가 어려우면 간격과 버튼 크기를 먼저 축소

### 태블릿 (`md ~ lg`)

- 1열에서 2열 또는 2분할 레이아웃 전환 허용
- 제목은 데스크톱형 크기로 확장 가능하나 2줄 이상 깨질 때 line-height 확인
- 목록형 대시보드는 카드 폭이 너무 넓어지지 않도록 `max-w` 유지

### 데스크톱 (`lg+`)

- 랜딩/대시보드만 넓은 `max-w-7xl`, `max-w-[1440px]` 사용
- 긴 여백과 다열 정보 밀도 허용
- 모바일 전용 축약 UI를 데스크톱까지 끌고 가지 않는다

## 페이지별 합격 기준

### P-001 랜딩 (`/`)

- 모바일에서 히어로 제목이 화면 밖으로 잘리지 않는다
- 역할 카드 2개는 모바일 1열, 태블릿 2열
- 카드 내부 라벨/설명/화살표가 같은 카드 경계 안에 남아 있어야 한다

### P-002 교사 자료 등록 (`/teacher/upload`)

- 모바일에서 페이지 제목과 폼 첫 입력이 모두 첫 화면 또는 짧은 스크롤 안에 보여야 한다
- 완료 Alert의 액션 버튼 2개는 모바일에서 줄바꿈 또는 세로 스택 허용
- 파일 리스트 상태 아이콘/텍스트가 한 줄에서 깨져도 정보는 유지되어야 한다

### P-003 학생 질문 (`/student/ask`)

- 고정 헤더와 하단 입력바 사이에서 메시지 영역이 충분히 보여야 한다
- 모바일에서 모드 선택 3개와 전송 버튼이 모두 화면 안에 들어와야 한다
- textarea placeholder, 입력 내용, 전송 버튼이 가로로 잘리지 않아야 한다
- 마지막 메시지와 추천 보기 영역이 하단 바에 가려지지 않아야 한다

### P-004 교사 대시보드 (`/teacher/dashboard`)

- 모바일에서는 통계/히트맵/질문 로그/추천 카드가 1열로 자연스럽게 쌓여야 한다
- 대시보드 제목과 수업명은 잘리지 않아야 한다
- 표 형태는 모바일에서 가로 스크롤이 필요하면 카드형 또는 `overflow-x-auto`로 명시 처리

### P-005 학생 요약 (`/student/summary`)

- 모바일에서 요약 카드, 다룬 개념 리스트, CTA 버튼 2개가 세로로 자연스럽게 쌓여야 한다
- 고정 하단 네비게이션이 있다면 CTA를 가리지 않아야 한다

## 구현 가이드

### 타이포

- 페이지 제목: `text-4xl md:text-[3.5rem]`처럼 단계형 축소를 기본으로 한다
- 매우 긴 제목은 `clamp(2.25rem, 8vw, 3.5rem)` 허용
- `text-[10px]` 마이크로 라벨은 유지하되, 모바일에서는 주변 패딩을 먼저 줄인다

### 레이아웃

- 카드 그리드: `grid-cols-1 md:grid-cols-2 lg:grid-cols-*`
- 대시보드: `grid-cols-1 lg:grid-cols-12`
- 랜딩 역할 카드: `grid md:grid-cols-2`

### 고정 바

- 상단 고정 헤더 높이가 `h-16`이면 본문 `pt-24` 이상 유지
- 하단 고정 입력바가 있으면 본문 `pb-40 ~ pb-48` 유지
- 모바일 입력바 기본값:
  - 컨테이너 `gap-2` 권장
  - 전송 버튼 `w-12 h-12` 우선 검토
  - textarea 최소 높이 `min-h-12 ~ min-h-14`

### 금지

- 모바일에서 `text-[3.5rem]` 고정 사용
- `style={{ borderRadius: 0 }}` 같은 인라인 보정으로 반응형 이슈를 덮기
- 모바일에서 3개 이상의 핵심 조작 요소를 너비 계산 없이 한 줄 고정

## QA 체크리스트

### 모바일 390px

- 헤드라인이 잘리지 않는가
- 본문에 가로 스크롤이 없는가
- 하단 고정 바가 입력/버튼/마지막 콘텐츠를 가리지 않는가
- 주요 CTA를 한 손가락으로 누르기 쉬운가

### 태블릿 768px

- 컬럼 전환이 어색하게 비거나 겹치지 않는가
- 헤더/본문/푸터의 최대 폭이 과도하게 좁지 않은가
- 표/카드/그리드가 1열에 머무르지 않고 적절히 확장되는가
