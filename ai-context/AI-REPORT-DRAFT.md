## 1. 기획

### ■ 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

**사용자:** 중·고등학교 수학 교사 + 학생 (방과 후 자기주도 학습 상황)

**문제점 4가지:**

1. **학생은 수업 후 막히지만 질문을 정리하기 어렵다**
   - KCI 등재 연구(2025): 질문 저해 요인 — 학습 환경, 부정적 평가 걱정, 자신감 부족, 질문 동기 부족
   - "질문이 중요하다는 인식" vs "실제 질문이 활발하지 않음" 간 구조적 괴리 존재

2. **학생은 정답만 보고 넘어가서 진짜 이해가 안 된다**
   - 콴다(QANDA) 등 기존 도구는 정답을 바로 제공 → 교육적 가치 논란
   - Brookings 연구: AI 튜터가 정적 스캐폴딩에 의존하면 AI 의존성 증가 위험

3. **교사는 누가 어디서 막히는지 한눈에 보기 어렵다**
   - OECD TALIS 2024: 한국 교사의 주당 행정 업무 시간이 OECD 평균 초과
   - 개별 학생의 오개념을 파악하려면 1:1 면담이 필요하나 현실적으로 불가능

4. **교사는 같은 설명을 반복하느라 시간이 많이 든다**
   - 30명 학급에서 같은 개념을 개별 설명하는 비효율
   - AI 디지털교과서(AIDT) 2학기 채택률 58.8% 급감 → "수업 후 보충" 영역은 여전히 공백

### ■ 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

**핵심 기능 4가지:**

1. **Grill-Me 질문 사다리 (AI가 답변이 아닌 질문을 던짐)**
   - Matt Pocock의 grill-me 패턴 + 소크라틱 방법론을 학습에 적용
   - AI가 4단계로 질문을 던지고, 학생이 스스로 답을 찾도록 유도
   - 매 질문에 접이식 "추천 보기" 제공 (학생이 완전히 막히면 참고)
   - 근거: Khanmigo 실증 — 학생 68%가 정답형(ChatGPT)보다 질문형(소크라틱) 선호

2. **수업 자료 기반 응답 (Context Stuffing + 확장성)**
   - 교사가 업로드한 PDF/Obsidian .md 자료에 근거해서만 응답
   - MVP: 자료 전체를 128K 컨텍스트에 주입 (빠르고 정확)
   - v2: pgvector 벡터 검색으로 자동 전환 (스키마 미리 준비됨)
   - 근거 없는 응답에는 "⚠️ 수업 자료에서 확인되지 않습니다" 명시

3. **교사용 오개념 대시보드**
   - 학생 질문 로그를 AI가 배치 분석 → 자주 막히는 개념 TOP 5 히트맵
   - 보충 설명 추천 카드 제공
   - 디자인 4원칙 적용: Simplify → Prioritize → Contextualize → Integrate

4. **질문 의도 자동 분류**
   - 학생 질문을 4가지로 자동 분류: 개념 설명 / 풀이 힌트 / 답안 검토 / 유사 문제
   - 분류 결과에 따라 다른 프롬프트 전략 적용 → 토큰 효율화

### ■ 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

**정량적 근거:**

| 지표 | 근거 | 출처 |
|------|------|------|
| 수학 성취도 향상 | **0.23 SD** (50th→59th percentile) | Khanmigo/WestEd 2024 |
| 대수 준비도 향상 | **0.15 SD** | Khanmigo/WestEd 2024 |
| 영어학습자 수학 성취 | **0.31 SD** (형평성 효과) | Khanmigo/WestEd 2024 |
| 소크라틱 방식 선호도 | 학생 **68%** | Khanmigo 사용자 조사 |
| 높은 스캐폴딩 효과 | **유의미한 개념 학습 향상** | ScienceDirect 2025 |

**정성적 기대 효과:**

- **학생:** 질문 장벽 해소 (익명 + AI 대화), 메타인지 활성화 (Grill-Me), 자기주도 학습 습관 형성
- **교사:** 오개념 즉시 파악 (히트맵), 반복 설명 시간 절감, 데이터 기반 보충수업 설계
- **학교:** AIDT 보완재로 "수업 후 보충" 공백 해결, API 비용 $0 (Gemma 4 로컬)
- **프라이버시:** 학생 데이터 외부 전송 없음 (로컬 모델), 개인정보 최소 수집

## 2. AI 활용 전략

### ■ 이 프로젝트에서 사용할 AI 도구와 모델은 무엇이며, 선택한 이유는 무엇인가요?

| 도구/모델 | 역할 | 선택 이유 |
|----------|------|----------|
| **Claude Code (Claude Opus 4.6)** | 전체 개발 협업 — 기획, 문서화, 코드 생성, 디버깅, 배포 | 1M 컨텍스트로 전체 프로젝트를 한 세션에서 관리. SSOT 문서 구조로 에이전트와 일관된 협업 가능 |
| **OpenAI GPT-4o-mini** (공모전) | 학생 튜터링, 의도 분류, 교사 요약 | 10만원 크레딧 보유, 안정적 스트리밍, 한국어 수학 품질 우수 |
| **Gemma 4 E4B** (프로덕션) | 로컬 튜터링 (노트북/폰) | AIME 89.2%, Apache 2.0, 5GB(4-bit), API 비용 $0, 프라이버시 보장 |
| **Vercel AI Gateway** | LLM 라우팅, OIDC 인증, 비용 추적 | provider 무관 모델 스위칭, 무정전 fallback |
| **Supabase** | DB + Storage + 벡터검색 인프라 | pgvector 내장, 무료 티어, 확장성 |

### ■ 각 AI 도구별 활용 전략을 작성해주세요.

**1. Claude Code — 에이전트 협업 전략**

| 항목 | 전략 |

**2. GPT-4o-mini / Gemma 4 — 튜터링 AI 전략**

| 항목 | 전략 |
|------|------|
| **페르소나** | 소크라틱 수학 튜터 — 답변 대신 질문을 던짐 (Grill-Me 방법론) |
| **핵심 제약** | 정답 직출 금지, 수업 자료 근거 필수, 4단계 질문 사다리 |
| **에이전트 구성** | 의도 분류기 → 자료 기반 응답기 → Grill-Me 질문 생성기 → 교사 요약기(배치) — 4개 역할 분리 |
| **데이터 흐름** | 학생 질문 → 의도 분류(JSON) → 수업 자료 컨텍스트 주입 → Grill-Me 프롬프트 → 스트리밍 응답 |
| **하이브리드 라우팅** | Task Router(JS 규칙)가 단순 질문은 Gemma 4 로컬, 복잡 질문은 Cloud API로 분배 |
| **Fallback** | WebGPU 미지원 기기 → 자동 Cloud 전환, API 오류 시 → 캐시된 응답 제공 |
